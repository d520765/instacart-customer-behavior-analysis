# instacart-customer-behavior-analysis
Полный анализ поведения покупателей Instacart с использованием Python, SQL и Power BI: подготовка данных, EDA, анализ покупок, повторяемости, категорий и интерактивный дашборд.

![dashboard](./dashboard.png)

Основные цели проекта
- Провести загрузку и очистку данных Instacart  
- Построить витрину данных (факт-таблицу) для SQL и Power BI  
- Выполнить исследовательский анализ (EDA)  
- Проанализировать поведение пользователей  
- Изучить повторяемость покупок  
- Определить топовые категории и товары  
- Визуализировать результаты в Power BI  
- Сформулировать бизнес-рекомендации

Используемые инструменты

Python 
- pandas, numpy  
- matplotlib  
- очистка данных  
- объединение таблиц  

SQL
- агрегаты, фильтры, группировка  
- топ товаров  
- активность по времени и дням недели  
- анализ повторных покупок

Power BI
- интерактивный дашборд  
- измерения и калькуляции DAX  
- фильтры по товарам/категориям  
- визуализация поведения пользователей


Основные инсайты

- Пиковые часы заказов — с 9 до 15 часов, максимум около 11:00  
- Наиболее загруженные дни — 0 и 1 день недели (суббота и воскресенье)  
- Категории produce, dairy eggs и beverages генерируют >50% заказов  
- Самый частый товар — Bananas
- Повторяемость ~59% говорит о высокой лояльности пользователей  
- Средний пользователь делает >1 повторной покупки → Instacart удерживает клиентов


Бизнес-рекомендации

- Укреплять ассортимент ключевых категорий (produce, dairy eggs)  
- Продвигать товары с высокой повторяемостью (bananas, avocados, berries)  
- Использовать персонализированные рекомендации в утренние часы  
- Проводить акции в пиковые дни  
- Развивать cross-selling в категориях с низкой повторяемостью  
- Тестировать программы лояльности, повышающие повторные заказы
